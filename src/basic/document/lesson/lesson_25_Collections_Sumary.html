Java Collection Summary

Hôm trước chúng ta đã đi qua các Interface, các Classes của Collections Framework rồi.

Hôm nay ta sẽ đi chi tiết hơn về chúng nó để xem cách sử dụng, các điểm cần ghi nhớ nhằm tổng hợp lại cho dễ nhớ mà thôi

Dựa vào các Interface, các Abstract Class, ta chia nó thành 2 nhóm chính.

  • List Interface : Array List, Linked List, Vector
  • Set Interface : Hash Set, Linked HashSet, Tree Set.
  • Map Interface : Hash Map, Linked Hash Map, Tree Map, Hash Table

How Hash Map works in Java

Trên đây, ta đã tìm hiểu qua về hash map rồi. Giờ cùng nhau đi sâu xem nó hoạt động như thế nào nào.

Ta đều đã biết, hash map hoạt động dựa trên nguyên tắc băm (hashing principle). Hashing là cơ chế chỉ định mã duy nhất cho một biến hoặc thuộc tính sử dụng một số thuật toán truy xuất dễ dàng. Một cơ chế hashing phải luôn return cùng 1 hashCode() khi mà nó được áp dụng cho cùng 1 đối tượng.

Vậy, hashing có liên quan gì đến việc lưu trữ và lấy ra value ở trong HashMap? Thường câu trả lời sẽ là value sẽ được lưu trữ trong các bucket, và việc lấy ra dựa trên key. Tuy nhiên đấy là điều hoàn toàn sai nhé.

Chúng ta đều biết rằng, để lấy ra các Value - Object được lưu trữ bên trong Hash Map, ta làm việc với Class Entry. Đấy chính là vấn đề. Nào, hãy cùng nhìn sâu vào bên trong Hash Map xem nhé.

/**
* The table, resized as necessary. Length MUST always be a power of two.
*/
transient Entry[] table;

Rõ rồi chứ, Hash Map nó lưu trữ đối tượng giống như cá Entry instances, chứ không phải dưới dạng key - value.

Entry Class

Hash map có 1 lớp bên trong gọi là Entry Class, để giữ key và value

static class Entry<K,V> implements Map.Entry<K,V> {
    final K key;
    V value;
    Entry<K,V> next;
    final int hash;
    ........
}

Và giờ, ta nhận ra rằng, Hash Map lưu trữ các Entry instance dưới dạng là một array (Entry[] table không phải là từng cặp key - value. Để lưu trữ một giá trị, ta cần phải sử dụng put method của hash map. Giờ thì cùng xem xem cái method đó hoạt động ntn?

How put() method works internally
public V put(K key, V value) {
    if (key == null) return putForNullKey(value); // 1

    int hash = hash(key.hashCode()); // 2
    int i = indexFor(hash, table.length); // 3
    for (Entry<K,V> e = table[i]; e != null; e = e.next) { // 4
        Object k;
        if (e.hash == hash && ((k = e.key) == key || key.equals(k))) { // 5
            V oldValue = e.value;
            e.value = value;
            e.recordAccess(this);
            return oldValue;
        }
    }
    modCount++;
    addEntry(hash, key, value, i);
    return null;
}
  1. Đầu tiên, nó kiểm tra xem nếu key có null hay ko. Nếu null thì nó sẽ store tại vị trí 0, vì hash code của null là 0.
  2. Sau đó nó applies hashCode với key.hashCode() bằng việc call hash code method.
  3. indexFor method được sử dụng để lấy chính xác location để store Entry Object
  4. Ở bước này, nếu như 2 object khác nhau có cùn một hash code, thì nó sẽ được lưu trữ trong cùng 1 bucket. Để xử lý điều này, chúng ta hãy nhớ lại cái LinkedList structure. Nó sẽ có 1 thuộc tính để chỉ tới 1 đối tượng tiếp theo. Tương tự như vậy, next attribute của entry sẽ chỉ tới Object tiếp theo. Sử dụng các object khác nhau có cùng một hash code sẽ được đặt cạnh nhau.
  5. Trong trường hợp này, Hash Map sẽ kiểm tra giá trị của next attribute kia. Nếu nó là null, thì Value này sẽ được lưu tại đó, còn nếu ko, nó sẽ tiếp tục duyệt tiếp để tìm vị trí thích hợp bỏ vào.

    ** Lưu ý cái dòng số 4 nhé, nó duyệt qua các Entry của Map để tìm vị trí còn trống.
How duplicate key is prevented in hashmap ?

Chúng ta đều đã biết, Hash map không cho phép duplicate key. Khi chúng ta insert các key giống nhau nhưng value khác nhau, thì chỉ có giá trị cuối cùng mới được trả về.

Ta xem ở dòng số 5 trên kia, nếu như nó kiểm tra cái hash code truyền vào mà equal với cái hash code trong bucket hiện tại, thì nó sẽ thay thế giá trị cũ bằng giá trị mới. Vì vậy, nó ngăn chặn việc xuất hiện 2 object mà cùng 1 key xuất hiện ở đây.

How get() method works internally ?
public V get(Object key) {
    if (key == null) return getForNullKey(); // 1
    int hash = hash(key.hashCode()); //2.
    for (Entry<K,V> e = table[indexFor(hash, table.length)];e != null;e = e.next) { // 3
        Object k;
        if (e.hash == hash && ((k = e.key) == key || key.equals(k))) // 4
        return e.value;
    }
    return null;
}

Phương thức này có nhiều nét tương đồng với cái get trên kia. Cùng duyệt qua các bước của nó nhé.

  1. Đầu tiên, nó cũng kiểm tra xem key có null hay không, nếu null thì nó trả về cái vị trị 0 mà lúc trước nó set vào
  2. Tiếp đến nó sẽ lấy hash code của key đang kiểm tra.
  3. Sau đó, nó tìm tới cái bucket location của cái key đó.
  4. Tại đây, nó duyệt các phần tử trong cái table (từ vị trí của hash code đến cuối), để tìm xem có cái entry nào có cùng hash code với key và equal với key ko, nếu có thì nó sẽ trả lại cái value - Object của entry đó đang giữ, nếu ko thì trả lại null.
What happens if two keys have the same hashcode?

Thế lỡ điều gì xảy ra nếu như 2 key có cùng hash code? Xem ở dòng số 4 trên kia, ta thấy nó ngoài việc kiểm tra xem hash của Entry bằng hash của key đang kiểm tra, nó còn check xem key đang kiểm tra và key của entry có bằng nhau hay ko. Vì vậy nó luôn trả lại đúng giá trị mà nó đã push vào.
