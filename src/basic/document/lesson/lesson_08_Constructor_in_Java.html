Constructor and Main Function in Java

Hôm nay chúng ta sẽ tìm hiểu về 2 cái khá quan trọng trong Java, đó là constructor và main function.

Constructor in Java and Types of Constructors in Java

Constructor is a special method in Java which is used to initialize the object. It looks like a normal method however it is not. A normal java method will have return type whereas the constructor will not have an explicit return type. A constructor will be called during the time of object creation (i.e) when we use new keyword follow by class name.

Constructor là một method đặc biệt trong Java, được dùng để khởi tạo object. Nó trông có vẻ giống như một phương thức bình thường, tuy nhiên nó ko phải giống như vậy. Những method bình thường sẽ có kiểu trả về rõ ràng còn cái constructor này thì ko. Nó được gọi trong thời gian khởi tạo đối tượng, khi ta sử dụng từ khóa new theo sau bởi tên lớp.

Rules for creating a Constructors in Java

Có 2 rulers quan trọng cần nhớ trước khi tạo một constructor

  • Constructor phải có tên trùng với tên của class name
  • Constructor không cần phải có kiểu dữ liệu trả về rõ ràng.

Types of Java Constructors

  • Default Constructor (or) no-arg Constructor
  • Parameterized Constructor
Can a Constructor return any value ?

Constructor không trả về một kiểu dữ liệu rõ ràng, nhưng mặc định, nó trả về thể hiện của lớp.

Car() {     System.out.println("Default Constructor of Car class called");
}

public static void main(String args[]) {
    Car c1 = new Car();
}
Java – Constructor in an Interface?

Không. Chúng ta không thể có constructor nào được định nghĩa trong một interface. Những method trong Interface luôn được khai báo public và abstract (mặc định) để cung cấp 100% trừu tượng, và việc thực hiện - implements(implementation methods) sẽ được cung cấp bởi class implement nó.

Vậy, tại sao ko thể tạo Constructor cho Interface trong Java???

Trước hết, để có thể gọi một phương thức, ta cần có 1 object. Tuy nhiên, phương thức trong interface lại ko có body, vì vậy không cần thiết để gọi những method trong interfaces. Vì chung ta không thể gọi các phương thức trong interface, nên chúng ta không cần thiết tạo các object để làm cái gì cả. Và vì không cần tạo object, nên cũng chả cần khai báo constructor cho nó làm gì.

Constructor phải thuộc về những class mà implement nó. Một interface, chỉ đóng vai trò như một bản hợp đồng - contract cho class mà implemenet nó.

Can we have both Default Constructor and Parameterized Constructor in the same class?

Tất nhiên là có, JVM sẽ tự động phân tích để biết nên gọi constructor nào. Và ko những thế, còn có thể tạo nhiều constructor cho class, miễn là nó khác tham số đầu vào.

Can i have a class with No Constructor ? What will happen during object creation ?

Tất nhiên là có. Khi một class không khai báo bất cứ một Constructor nào, thì tại thời gian object được khởi tạo, compiler sẽ tự động tạo ra một default constructor cho nó.

Does compiler create Default constructor everytime ?

Không, nó sẽ không tạo khi mà bạn đã có một constructor rồi. Giả sử ta có một Class Car với Constructor là Car (String color), khi ấy nếu ta gọi new Car() thì nó sẽ báo lỗi. Default constructor không được tạo trong trường hợp này, vì đã có khai báo một constructor khác rồi.

Java Constructor Chaining

Constructor Chaining is nothing but calling one Constructor from another. We will be using this keyword and super keyword in calling a constructor. this can be used to call a constructor within the same class whereas super can be used to call the constructor of the Parent class.

Cái này đề cập đến việc dùng this và super trong Constructor. Trong constructor ta cần sử dụng this để cho biết rằng việc set giá trị của method này là áp dụng cho các instance variable của object.

super keyword được sử dụng trong các class con kế thừa từ thằng cha. Khi đó super là chỉ tới thằng cha của nó

What is the use of a Private Constructors in Java

Thông thường, Constructor của các class đều được khai báo là public để các lớp khác có thể sử dụng và tạo ra object của class đó. Tuy nhiên, trong một số trường hợp, ta muốn quản lý/ kiểm soát điều này, ta sẽ sử dụng từ khóa private để ngăn chặn việc tạo object từ các class bên ngoài. Việc khởi tạo chỉ được thực hiện bên trong class và ta có thể kiểm soát chúng.

Khi nào nên sử dụng nó: Khi muốn kiểm soát việc tạo instance cho class

Ví dụ : Singleton.

Main function in Java

Hàm main trong hầu hết các ngôn ngữ lập trình là hàm đầu tiên mà chúng ta sẽ làm quen khi học một ngôn ngữ mới. Nó đóng vai trò như một điểm bắt đầu (Entry point), nơi mà quyền điều khiển được chuyển từ hệ thống đến một chương trình máy tính, và tại đó, CPU sẽ truy cập vào một chương trình hay một đoạn code và tiến hành thực thi.

main có phải là một reserved keyword trong Java không?

Không, ta có thể khai báo một hàm main khác, hoặc khai báo một biến với giá trị giống nó. ex int main = 100;

Hàm main có cần modifier là static không?

Hàm main cần có modifier là static để cho JVM có thể dễ dàng tìm ra nó mà không cần phải chạy constructor của class. Hàm main mà ko có static modifier sẽ gây ra tính mơ hồ cho JVM bởi vì những hàm ko phải static cần được gọi thông qua constructor và JVM sẽ không biết phải gọi constructor của Class nào.

Tóm lại luôn phải khai báo static cho nó để đánh dấu nó là entry point cho JVM biết và khởi động.

Tham số String[] args có bao giờ null được không?

Không bao giờ. Kể cả khi bạn ko có gì để truyền vào thì đơn giản chỉ cần truyền vào một chuỗi rỗng là được.

Có thể viết là static public void main thay vì public static void main được không?

Vị trí của modifier như public hay static ko quan trọng nên khai báo thế nào cũng được. Nhưng return type thì quan trọng nên luôn phải khai báo void trước main vì lý do đó.

Có thể overload hàm main được không?

Có, nhưng vô nghĩa. JVM chỉ gọi hàm main static với tham số truyền vào là String []. Vì vậy không thể Overload nó với mục đích thay thế entry point mặc định cho ứng dụng

JVM luôn xem hàm main với khai báo modifier là static và parameter truyền vào là String array, nên đâu có cách nào overload được hàm main để biến nó thành một entry point khác đâu.

Có thể dùng tên hàm khác thay vì main không?

Không, cũng như cái câu hỏi Overloading ở trên, nó vô nghĩa.
