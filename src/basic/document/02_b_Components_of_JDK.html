Components of Java Development Kit (JDK)

JDK có rất nhiều tool mà những Java deveper cần dể deployed Java Program và JRE để chạy chương trình. Chúng ta có thể định nghĩa đơn giản rằng JDK = JRE và Development Tools. Development Tools bao gồm:

  • Java Archive (jar)
  • Java Compiler (javac)
  • Java Disassembler (javap)
  • Java Debugger (jdb)
  • Java Header File Generator (javah)
  • Documentation (javadoc)
  • và nhiều thằng nữa.

Trong bài này, chúng ta chỉ đi xem xét những thằng phổ biến thôi.

Java Archive (jar)

Java Archive (jar) tool combines multiple Java files/ applets into a single archive .jar (Java Archive).

jav là một công cụ lưu trữ và nén dựa trên các định dạng ZIP hoặc ZLIP

Mục đích chính của nó là được thiết kế cho packet applets hoặc multiple files vào 1 file lưu trữ

Những files, image, media .... sẽ được combined vào 1 file, giúp cho mấy thằng khác có thể download về và sử dụng

Vì nó được nén lại thành 1 nên việc download nó về cũng nhanh hơn vì chỉ mất 1 lần, và cải thiện tốc độ vì file đã được nén lại.

Để né file (tạo Jar file), sử dụng câu lệnh
jar cf <<jar filename>> <<input file(s)>>

Giải thích thêm về những option ở trên kia.

  • ‘c’ – This indicates that we want to create a jar file.
  • ‘f’ – This indicates that we need the output to goto a file rather than sdout.
  • jar filename – The name of the jar which needs to be created, the filenames are given with the extension of ‘.jar’ as a convention though it is not a mandatory.
  • input files – List of the files which you want to put inside the jar file, multiple files can be added each separated by a space
Để xem content bên trong file, sử dụng câu lệnh
jar tf <<jar filename>>

Giải thích thêm về những option ở trên kia.

  • ‘t’ – This indicates that you want to view the contents of the jar file
  • ‘f’ – This indicates that the JAR file whose contents are to be viewed is specified on the command line.
  • jar filename – The name of the jar whose contents needs to be viewed
Để giải nén (extracting the content of a JAR file), sử dụng câu lệnh
jar xf <<jar filename>>
  • ‘x’ – This indicates that you want to extract the contents of the jar file
  • ‘f’ – This indicates that the JAR file whose contents has to be extracted is specified on the command line.
  • jar filename – The name of the jar whose contents needs to be extracted
Để update Jar file
jar uf <<jar filename>> <<input file(s)>>
  • ‘c’ – This indicates that we want to update an existing jar file.
  • ‘f’ – This indicates that the JAR file whose contents has to be updated is specified on the command line.
  • jar filename – The name of the jar whose contents needs to be updated.
  • input files – List of the files which need to be added in addition

Java Compiler (javac)

Mục đích của nó là compiler các file *.java thành *.class

Javac là compiler sử đụng trong Java. Vị trí của Javac tool là ở trong folder /bin bên trong thư mục cài đặt JDK. Java Compiler đọc toàn bộ thông tin class, interface và những code khác được viết bởi ngôn ngữ lập trình java và compiler (phiên dịch) nó thành byte code (.class files).

Có 2 cách để convert source code file bằng javac

javac <<file1.java>> <<file2.java>>

Nếu có nhiều file quá, thì create bằng command line sau:

javac @filelist

Java Disassembler (javap)

Mục đích của nó là disassembles các file *.class thành *.java

Javap cũng như Javac đều được lưu trong folder /bin trong thư mục cài đặt JDK. Khác với thằng Javac compiler cái file java thành bytecode, thằng này tháo dỡ (disassembles) một hoặc nhiều class mà truyền vào nó. Javap command in ra các packet, protected và public fields cũng như những method của class mà truyền vào trong nó. Javap command int ra output trong stdout.

javap <<.class file1>> <<.class file2>>

Thực ra cái thằng này chả in ra được như thế đâu. Những gì nó xuất ra chỉ đơn giản là :

Compiled from "Sample.java"
public class Sample{
    java.lang.String name;
    public Sample(java.lang.String);
    public void show();
    public static void main(java.lang.String[]);
}

Đơn giản nó chỉ show lên các thông tin mà bên ngoài được phép truy cập vào thôi.

Java Debugger (jdb)

Java Debugger (jdb) is a command-line debugger for Java classes it helps us in debugging our Java code. In order to start debugging we simply need to give the class name after jdb

jdb <<class name>>

Đơn giản cái này là cái công cụ debug.

Java Header File Generator (javah)

Mục đích của cái này là generate ra C header và source file (.head).
