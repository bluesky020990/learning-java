Interface and Abstract in Java

Trong bài học này, chúng ta sẽ đi sâu vào 2 cái thằng gọi là quan trọng nhất trong java đối với tính trừu tượng - abstract and interface.

Abstract Class in Java

Từ khóa abstract chỉ có thể sử dụng cho class và các method trong Java. Một phương thức được khai báo là abstract, thì bản thân nó sẽ không được triển khai, mà trách nhiệm đó được đẩy về cho các Class mà extend nó. Vì vậy, abstract method thì ko cần implement. Luôn nhớ điều này.

Hãy đi sâu vào để xem chi tiết hơn nhé.

Khi một class được khai báo là trừu tượng, thì những lớp đặc biệt này sẽ ko được khởi tạo. Nó chỉ có thể được extend và những phương thức bên trong nó phải được implement bởi class extend nó.

Những điểm cần lưu ý về Abstract Class:

  • Abstract class có thể có cả phương thức abstract và non-abstract
  • Một phương thức trong class được khai báo là trừu tượng, thì lớp đó cũng bắt buộc phải khai báo trừu tượng
  • Nếu một phương thức được khai báo trừu tượng, thì nó sẽ không phải implement bất cứ cái gì. Nếu có sẽ báo lỗi. Abstract class và abstract method như một bộ xương vayak, nó defined structure, mà không có bất ứ implementation nào.
Can Abstract Class have constructor?

An abstract class can have constructor, abstract method, non abstract method , data member and even main method as well.

Có, nhưng new 1 Abstract Object thì đồng nghĩa với việc phải override lại các function của nó.

public abstract class AbstractClass {
    abstract void run();

    abstract void eat();

    abstract void sleep();

    public AbstractClass() {
        System.out.println("This is constructor of abstract class");
    }
}
Important points to remember
  • If we have declared a method as abstract then you must declare the class as abstract, abstract method cannot be present in a non-abstract class.
  • Abstract Class can have concrete (cụ thể) methods(non-abstract) as well .
  • Abstract method should not have implementation(no body).
  • Abstract method declaration should end with a semicolon (;)
  • The class extending the abstract class should implement all the abstract methods.
Interface in Java and Uses of Interface in Java

Abstraction là quá trình mà class che dấu những chi tiết thể hiện, chỉ hiển thị chuwcsn năng cho User. Trước đó thì ta đã tìm hiểu qua về abstract, giờ chúng ta lại đi qua thằng Interface, một trong số những cách để đạt được abstract 100%.

What is an Interface?

An interface is a blueprint of a class, it can have methods and variables like a class but methods declared in a interface will be by default abstract(only declaration no body) and the variables declared will be public, static & final by default.

Use of Interface in Java
  • Using interface we can achieve 100% abstraction in Java, as the methods doesn’t have body and the class needs to implement them before they access it.
  • Java does not support multiple inheritance using the interface we can achieve this as a class can implement more than one interface.
Can Abstract Class have constructor?

Khác với abstract class - có thể có constructor, Interface không thể có constructor, cũng ko thể có non-abstract method và main method. Tóm lại nó như một tờ giấy khai báo, chỉ là cái biểu mẫu, chứ ko được sử dụng cho mục đích khác. Cái mẫu hóa đơn thì làm sao mà trở thành cái hóa đơn được, đúng ko?

Important points to remember
  • An Interface cannot be instantiated in Java.
  • Methods declared in a interface should be public and abstract
  • Interface cannot have concrete methods(non-abstract methods or methods with body)
  • variables declared should be public, static & final even though if you miss any one or all the qualifiers it will be automatically assigned.
  • Interface variables must be initialized at the time of declaration else the compiler will throw error.
  • An Interface can only extend other interface (only one)
  • A Class can implement any number of interface
  • If two interface have methods with same signature and same return type then the implementing class can implement any one of those
  • If two interface have methods with same signature and different return type cannot be implemented at the same time
  • In the implementing class we cannot change the variable value which is declared in the interface as it is final by default
  • Một interface thì ko thể khởi tạo trong Java. - Vì nó có hàm constructor quái đâu.
  • Method định nghĩa trong interface đều phải là public và abstract.
  • Interface ko có method cụ thể, tức là những method mà có body và không phải là non-abstrac. Mọi method của nó đều là abstract
  • Biến được khai báo trong interface (variable instances of Interface, not function) thì đều được khai báo là public final và static. Nếu không khai báo thì nó sẽ được thêm vào.
  • Interface variable thì phải được khởi tạo ngay tại thời điểm khai báo, nếu ko thì Compiler sẽ báo lỗi. Nói cách khác, ta không thể khai báo một biến int count mà không có giá trị được.
  • Một interface có thể extend từ 1 hoặc nhiều interface khác
  • Class có thể implement một hoặc nhiều interface
  • Nếu 2 interface có 2 method cùng tên, cùng variable và cùng kiểu return type thì Class implement nó tự quyền quyết định implement phương thức đó ntn. Nói cách khác, nó sẽ tự quyết định khi gọi phương thức đó thì nó sẽ hành xử như thế nào.
  • Nếu 2 phương thức trong 2 interface có cùng tên, cùng tham số nhưng khác kiểu dữ liệu trả về thì không thể implement trong cùng 1 thời điểm. (Class implement 2 interface trên kia không được, sẽ báo lỗi liền.
  • Những biến trong Interface đều khai báo là static và final nên ta không thể thay đổi chúng. (Biến của Interface)

    What is Marker Interface

    A Marker interface is a interface with no variables and methods, in simple words, we can say that empty interface in java is called marker interface. Serializable, Cloneable, Remote Interface are some of the examples of Marker Interface. In this article, we will discuss the uses of Marker interface in Java .

    Nói nôm na, Maker Interface là interface mà không có bất cứ variable và method nào. Nhưng nếu nó ko có cái đó thì nó dùng để làm gì???

    What is the use of Marker Interface?

    When we look closely the marker interface doesn’t have any methods in it, so then comes the question what is the use of it then? If you look closely Serializable, Cloneable, Remote Interface it looks like they will indicate something to compiler or JVM. So if the JVM sees Clonnable it performs some operation to support cloning. Same applies to the Serializable and other marker interfaces as well.

    Hiểu một cách đơn giản, tụi này cung cấp một số thông tin thiết yếu đến JVM để thực hiện một số hoạt động hữu ích

    Ví dụ như cái Clonnable Interface thì thực hiện một số hoạt động để hỗ trợ Nhân bản - Clone. Vì vậy, JVM đánh dấu những thằng này và biết rằng tụi class mà implement method này nó dùng vào mục đích gì.

    Why this indication cannot be done using a flag variable?

    Mục đích của Maker Interface là dùng để đánh dấu, vậy tại sao ta không sử dụng 1 biến flag để làm thay công việc này?

    Tất nhiên Flag nó vẫn có thể sử dụng thay thế cho trường hợp này, tuy nhiên việc sử dụng Maker Interface thì đơn giản hơn và dễ đọc hơn.

    Tuy nhiên, từ Java 1.5 thì cái thằng này thường ít được dùng cho mục đích này, sau khi Java giới thiệu một cái feature gọi là Annotation. Annotation là một lựa chọn tốt hơn là dùng Maker Interface để đánh dấu, vì chú thích thì có nhiều lợi thế hơn một giao diện. Lợi thế ở đây là code rõ ràng hơn, cách JVM đọc nó cũng dễ dàng hơn.
