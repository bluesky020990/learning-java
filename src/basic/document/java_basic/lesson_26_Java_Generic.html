Java Generic

Trong những bài trước, ta đã làm việc qua với nhiều kiểu dữ liệu khác nhau. Tuy nhiên hầu hết trong số chúng đều là cùng một kiểu Object.

Trong một vài trường hợp, ta muốn nó vừa xử lý kiểu đối tượng này, vừa xử lý kiểu đối tượng kia, vậy thì cần phải làm gì

Java cung cấp một biện pháp để giải quyết những vấn đề đó, nó được gọi là Generic (chung). Các ví dụ trong bài học này vui lòng xem tại GenericClassExample.class

Generic Methods

Chúng ta có thể viết một method mà nó có thể xử lý nhiều kiểu dữ liệu khác nhau, tùy thuộc vào tham số truyền vào. Compiler sẽ căn cứ vào từng đối tượng cụ thể để mà có thể xử lý từng phương thức một cách phù hợp.

Sau đây là các rule của nó:

  • Tất cả khai báo phương thức generic có một khu vực tham số kiểu được giới hạn bởi các dấu ngoặc nhọn (< và >) mà đứng trước kiểu trả về của phương thức ( < E > trong ví dụ sau đây).
  • Mỗi khu vực tham số kiểu chứa một hoặc nhiều tham số kiểu phân biệt nhau bởi dấu phảy. Một tham số kiểu, cũng được biết như là biến kiểu, là một định danh mà xác định một tên kiểu generic.
  • Các tham số kiểu có thể được sử dụng để khai báo kiểu trả về và hoạt động như là nơi giữ (placeholder) cho các kiểu của các tham số được truyền tới phương thức generic, mà được biết như là các tham số kiểu thực sự.
  • Phần thân phương thức generic được khai báo giống như bất kỳ phương thức nào khác. Ghi chú rằng các tham số kiểu chỉ có thể biểu diễn các kiểu tham chiếu, không phải là các kiểu gốc (như int, double, và char).

Xem ví dụ:

public static void testGenericMethod (){
    Integer [] listIntegerNumber = {10, 20, 30, 40, 50, 60, 70, 80};
    Long [] listDoubleNumber = {0l, 10l, 20l, 30l, 40l, 50l, 100l};

    displayValueOfArray(listIntegerNumber, "Show value of generic integer array ");
    displayValueOfArray(listDoubleNumber, "Show value of generic long array ");
}
// example 1
public static <E> void displayValueOfArray (E array[], E message){
    System.out.println(message);
    for(E element : array){
        displayValueOfElement(element);
    }
}
// example 2
public static <E> void displayValueOfElement (E element){
    System.out.println("This value is " + element);
}
1. Các parameter là generic param phải được khai báo bên trong dấu < >. Một method là Generic bắt buộc phải khai báo <E> ở đầu. Xem ví dụ 1 và 2.
2. Các parameter cách nhau bằng 1 dấu ,
3. Kiểu generic parameter này có thể được sử dụng làm kiểu trả về, hoặc cũng có thể được dùng để pass vào một generic method khác. Xem trong ví dụ 1.
4. Không được sử dụng các kiểu dữ liệu nguyên thủy đối với Generic Method.

Tóm lại, trong cái này, ta cần lưu ý, trong một method là Generic cần phải tuân thủ đủ những yêu cầu trên. Và trong trường hợp này, ta thấy kiểu dữ liệu mà ta làm việc nó chỉ là Kiểu Wrapper mà thôi. Trong trường hợp ta cần thao tác với các class phức tạp hơn, thì ta cần chuyển sang phần tiếp theo

Bounded Type Parameters - Tham số giới hạn

Ta thấy, ở ví dụ trên kia, tất cả những method mà ta có thể làm việc được với kiểu Generic là vô cùng ít ỏi, chỉ là mấy kiểu của Object Class mà thôi.

Vì vậy, trong một vài trường hợp, ta muốn giới hạn kiểu Tham số truyền vào, hoặc là quy chụp nó về một nhóm nào đó để sử dụng các method của nhóm đó, khi đó ta cần sử dụng bounded type parameter.

Để làm được điều này, ta khai báo thêm cho nó extend từ một cái nào đó. Ví dụ

public static <T extend Comparable<T>> .....

Generic Classes

Ta thấy ở cái bounded type parameters trên kia, cái parameters thường bị giới hạn bởi cái Interface mà nó implements thì mới được thêm 1 vài method. Trong trường hợp ta muốn thao tác với 1 list các Object khác nhau, thì ta cần làm gì??
