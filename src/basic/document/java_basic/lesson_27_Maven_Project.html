Maven Project

Xong phần Java căn bản rồi, giờ ta đi sang phần khác thôi.

Hôm nay ta sẽ tìm hiểu về Maven.

Maven Project

Dưới đây là một cấu trúc của Maven projects

Item Default
source code ${basedir}/src/main/java
Resources ${basedir}/src/main/resources
Tests ${basedir}/src/test
Complied byte code ${basedir}/target
distributable JAR ${basedir}/target/classes

Bản chất của Maven là một đống các file POM.xml. Mỗi module nó sẽ được liên kết với nhau bằng các thiết lập trong file pom. Vì vậy sẽ vô cùng đơn giản khi muốn tách/ import một cái gì đó vào.

Thêm vào đó, Maven còn cung cấp một repository plugin vô cùng lớn, và nếu như làm Java thì chắc cũng đã đủ xài. Quan trọng là google search ntn thôi.

Features of Maven

Maven đem lại rất nhiều lợi ích trong quá trình quản lý project. Cụ thể:

  • Consistent usage across all projects.
  • Dependency management including automatic updating.
  • A large and growing repository of libraries.
  • Extensible, with the ability to easily write plugins in java or scripting languages.
  • Instant access to new features with little or no extra configuration.
1. Nó mang lại sự thống nhất cho tất cả các dự án. Nó định ra cái gọi là POM.xml, mỗi project đều phải tuân thủ theo điều đó. Và đó là cơ sở cho sự thống nhất mà nó đã mô tả.
2. Tự động update những dependence. 3. Có một lượng lớn các thư viện 4. Có khả năng mở rộng 1 cách dễ dàng. 5. Bổ nghĩa cho cái trên thôi.

POM

POM là từ viết tắt của Probject Object Model. Đây cũng chính là những unit cơ bản của Maven

NÓ là một file XML, nằm trong thư mục của project/ module.

POM chứa thông tin về project và chi tiết các cấu hình khác nhau, được sử dụng bởi Maven để build project.

POM cũng chứa các mục tiêu (file war hay file ear sau khi build xong) và các pluggin.

Cách hoạt động của MAVEN như sau, nó đọc file POM, thu thập các thông tin cấu hình và executes - (build) - mục tiêu.

Dưới đây là một vài cấu hình mà có thể được xác định trong POM:

  • project dependencies
  • plugins
  • goals
  • build profiles
  • project version
  • developers
  • mailing list
POM Example

Before creating a POM, we should first decide the project group (groupId), its name (artifactId) and its version as these attributes help in uniquely identifying the project in repository.

Mỗi project, ta cần phải định ra cho nó một cái gọi là groupID, và tên của nó (artifactId)

<project xmlns="http://maven.apache.org/POM/4.0.0" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd">

<modelVersion>4.0.0</modelVersion>
<groupId>com.khanhtran.moimoi</groupId>
<artifactId>project</artifactId>
<version>1.0</version>

</project>

Mỗi một project thì cần có 1 file POM
